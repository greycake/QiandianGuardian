## 🚀 如何获取APK文件

### 方法1：使用GitHub Actions自动构建（推荐）

这是最简单的方法，不需要安装任何开发工具！

#### 步骤：

1. **创建GitHub仓库**
   - 访问 https://github.com/new
   - 创建一个新仓库（比如命名为 `QiandianGuardian`）
   - 不要初始化README

2. **上传代码到GitHub**

   在项目目录下打开命令行（PowerShell或CMD），执行：

   ```bash
   cd C:\Users\hw\.easyclaw\workspace\QiandianGuardian
   
   # 初始化Git仓库
   git init
   
   # 添加所有文件（gradle-wrapper.jar会自动下载）
   git add .
   
   # 提交
   git commit -m "Initial commit: 黔电卫士"
   
   # 添加远程仓库（替换YOUR_USERNAME为你的GitHub用户名）
   git remote add origin https://github.com/YOUR_USERNAME/QiandianGuardian.git
   
   # 推送到GitHub
   git branch -M main
   git push -u origin main
   ```

3. **等待自动构建**
   - 推送完成后，GitHub会自动开始构建
   - 访问：https://github.com/YOUR_USERNAME/QiandianGuardian/actions
   - 等待构建完成（大约2-3分钟）

4. **下载APK**
   - 构建完成后，在Actions页面点击构建成功的任务
   - 在页面底部找到 "Artifacts" 部分
   - 点击 `app-release` 下载ZIP文件
   - 解压ZIP，里面就是 `app-release.apk`

### 方法2：本地构建（需要Android Studio）

如果你有Android开发环境，可以这样构建：

1. **安装Android Studio**
   - 下载：https://developer.android.com/studio
   - 安装并配置Android SDK

2. **打开项目**
   - 启动Android Studio
   - 选择 "Open an Existing Project"
   - 选择 `C:\Users\hw\.easyclaw\workspace\QiandianGuardian`

3. **构建APK**
   - 在菜单栏选择：Build → Build Bundle(s) / APK(s) → Build APK(s)
   - 等待构建完成
   - APK位置：`app/build/outputs/apk/release/app-release.apk`

## 📲 安装到华为Mate X2

1. **将APK传输到手机**
   - 通过USB连接手机
   - 或通过微信、QQ等发送APK文件

2. **允许安装未知来源应用**
   - 打开手机设置
   - 进入"应用和通知" → "权限管理"
   - 找到"安装未知应用"
   - 允许你的文件管理器安装

3. **安装APK**
   - 在文件管理器中找到APK文件
   - 点击安装
   - 如果提示"安装未知应用"，点击"设置"并允许

4. **授予权限**
   - 首次打开应用会请求多项权限
   - 点击"允许"授予所有权限
   - 系统会提示设置为"默认通话应用"，点击"允许"

5. **开启拦截**
   - 打开"黔电卫士"应用
   - 打开"拦截贵州电话"开关
   - 打开"拦截贵州短信"开关

## 🔧 故障排查

### 拦截不生效？

1. **检查权限**
   - 打开手机设置 → 应用 → 黾电卫士 → 权限
   - 确保所有权限都已授予

2. **检查默认通话应用**
   - 打开手机设置 → 应用 → 默认应用 → 通话应用
   - 确保设置为"黔电卫士"

3. **检查自启动**
   - 打开手机设置 → 应用 → 应用启动管理
   - 确保"黔电卫士"允许自动启动

4. **重启手机**
   - 有时需要重启才能生效

### 构建失败？

1. **检查Gradle版本**
   - 确保使用Gradle 8.0+

2. **检查JDK版本**
   - 确保使用JDK 17或更高版本

3. **查看构建日志**
   - GitHub Actions页面有详细错误信息
   - 根据错误信息修复
